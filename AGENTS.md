# AGENTS.md

## 沟通规则

- 与用户沟通时使用**中文**。
- 在讨论实现细节、取舍和进度时，表达应简洁、清晰，并以执行为导向。

## 项目语言规则

本项目的所有交付内容默认以**英文**为主。

以下内容必须使用**英文**：

- 源代码
- 标识符
- 提交信息
- 代码注释
- README
- 文档
- CLI 帮助文本
- 日志和错误信息
- 测试名称
- 示例文件
- 规范说明文件

例外文件：

- `AGENTS.md`
- `TODO.md`
- `PLAN.md`

这三个文件使用**中文**编写。

## 计划相关文件

- 将可执行的待办事项写在 `TODO.md`
- 将整体计划、架构方向和阶段性思考写在 `PLAN.md`

规则：

- `TODO.md` 用于记录具体任务和清单式执行项
- `PLAN.md` 用于记录更高层的规划和项目整体方向
- 保持 `TODO.md` 与 `PLAN.md` 内容一致，但不要混淆它们各自的职责

## 执行规则

- 第一阶段不要引入不必要的抽象
- 严格遵守仓库中既定的命名与边界约束
- 优先选择稳定、明确的实现，而不是过早追求可扩展性
- 保持项目独立于具体渲染库
- 未经明确要求，不要添加引擎特定的集成

## 输出风格

在创建或修改项目文件时：

- 默认使用英文
- 保持命名一致
- 保持结构清晰、精简
- 优先考虑可维护性，而不是炫技式实现

在向用户汇报时：

- 使用中文
- 清楚说明改动了什么
- 说明重要的取舍和假设
